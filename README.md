# מערכת צירים — הרביע הראשון

אפליקציה אחת מאוחדת ללימוד מערכת הצירים ברביע הראשון: **51 עמודים ממוספרים** (דפי עבודה ו־8 שעשועונים משולבים), עמוד שער, תצוגת מובייל מצוינת והדפסת **A4** מקצועית — הכול בריפו אחד, בכתובת אחת ובקובץ כניסה אחד.

> ריפו יחיד מותר לפרויקט: `yanivmizrachiy/coordinate-first-quadrant`. אין לכתוב אל `yanivmizrachiy/parabula-next`.

## מה יש באפליקציה

- **דפי העבודה** — 51 עמודים מאורגנים לנושאים, עם ניווט עמוד־אחר־עמוד, תוכן עניינים, מסך מלא, שמירת המיקום האחרון והדפסת עמוד בודד.
- **החוברת המלאה** — עמוד השער ואחריו כל 51 העמודים, להדפסה כ־A4.
- **שעשועונים** — 8 שעשועונים, כל אחד עמוד ממוספר בתוך הנושא שלו (אין אזור נפרד), שנבנו כרכיבים מתמטיים מדויקים ולא כתמונות: מילת הסוד, ציור נסתר, המסלול המוצפן, הנקודה החשודה, אותו x או אותו y, כספת הקואורדינטות, מבוך הקואורדינטות, פענוח צבעוני.
- **מובייל** — RTL מלא, ללא גלילה אופקית, כפתורים גדולים, טעינה מהירה.
- **הדפסה** — כל עמוד וכל משחק ניתנים להדפסה עצמאית ב־A4, כולל תצוגת שחור־לבן חכמה.

## הרצה מקומית

```bash
npm install
npm run dev          # שרת פיתוח (Vite) עם רענון חם
```

## Build

```bash
npm run build        # בדיקת טיפוסים + בנייה ל-dist/
npm run preview      # תצוגה מקדימה של התוצר על http://localhost:4173
```

התוצר ב־`dist/` הוא אתר סטטי מלא — קובץ `index.html` אחד ונכסים, ללא שרת וללא תלות ברשת בזמן ריצה.

## בדיקות

```bash
npm test             # בדיקות מתמטיקה, משחקים ותוכן החוברת (Vitest)
npm run test:visual  # בדיקות תצוגה end-to-end (Playwright; דורש npx playwright install)
```

## פרסום ל־GitHub Pages

הפרסום **ידני בלבד** ואינו מתרחש אוטומטית. יש להריץ מתוך לשונית **Actions** את ה־workflow בשם *Deploy to GitHub Pages (manual)*. ה־`base` מוגדר יחסית (`./`) כך שהאתר עובד גם בכתובת של תת־נתיב ב־Pages.

## מבנה התיקיות

```
index.html                 קובץ הכניסה היחיד (Vite)
src/
  main.ts                  אתחול, סרגל עליון וניתוב
  router.ts                נתב מבוסס hash (מתאים ל-Pages)
  styles/                  מערכת עיצוב: tokens, base, app, workbook, grayscale
  lib/                     coordinateGrid (SVG), coordinateMath (טהור ונבדק), dom, storage
  data/workbook/pages/     קובץ אחד לכל עמוד, בשם שמתאר מה הוא מלמד
  data/workbook/authoring.ts  בניית הגיליון: כותרת, מספר, כותרת תחתונה, כלי כתיבה
  data/workbook/index.ts   BOOK — סדר הקריאה, ומכאן נגזרים המספור והנושאים
  games/                   8 המשחקים + מנוע "גילוי מילה/קוד" משותף
  views/                   מסכים: בית (שער + כפתורי פעולה), תוכן עניינים, עמוד, חוברת
public/assets/covers/      תמונת השער המאושרת
tests/                     בדיקות Vitest + בדיקות e2e של Playwright
docs/                      תיעוד אדריכלות, תוכן, משחקים, הדפסה, מובייל ובדיקות
```

## כיצד להוסיף עמוד

1. צרו קובץ חדש ב־`src/data/workbook/pages/` (שם שמתאר מה העמוד מלמד) והשתמשו ב־`sheet({ sectionClass, title, subtitle, content })`. כותבים רק את תוכן העמוד — הכותרת, מספר העמוד והכותרת התחתונה נבנים לבד. אין צורך לבחור מספר.
2. ייצאו אותו מ־`pages/index.ts` ושבצו אותו במקום הנכון ב־`BOOK` שב־`src/data/workbook/index.ts` — המספור ומפת הנושאים מתעדכנים לבדם.
3. גרפים נכתבים כ־`<div class="coordinate-grid grid-md" data-points='…' data-segments='…'>` — הרכיב מצייר אותם אוטומטית.
4. הריצו `npm test` — הבדיקות אוכפות את כללי `USER_MEMORY.md` (מספור, ניסוח, גודל סרטוטים ועוד).

## כיצד להוסיף משחק

1. צרו קובץ ב־`src/games/` המייצא `GameDefinition` (או השתמשו ב־`createRevealGame` למשחקי "פתרו ← גלו מילה/קוד").
2. שמרו את **נתוני** המשחק נפרדים מהתצוגה, כדי שאפשר יהיה לבדוק אותם.
3. רשמו את המשחק במערך `GAMES` שב־`src/games/index.ts`.
4. הוסיפו בדיקה ב־`tests/games.test.ts` שמוכיחה שהמשחק פתיר ושהתוצאה נכונה.

## זיכרון הפרויקט

- `USER_MEMORY.md` — **דף הכללים היחיד**: כל דרישות יניב, ב־15 ראשי פרקים.
- `CLAUDE.md` — שלט הפניה בלבד; אין בו כללים משלו.
- `docs/` — אדריכלות, אימות מתמטי, שעשועונים, הדפסה ומובייל.

# סקירת השעשועונים

כל שעשועון נבנה מחדש כרכיב מתמטי מדויק (HTML/CSS/SVG/TypeScript), לא כתמונה. לכל אחד: הוראות ברורות, פתרון חד־משמעי, תוצאה סופית ברורה, התאמה להדפסה, ובדיקה אוטומטית שמוכיחה שהוא פתיר ושהתוצאה נכונה (`tests/games.test.ts`).

| משחק | מזהה | מה מתרגל | תוצאה סופית | בדיקה |
|------|------|----------|-------------|-------|
| מילת הסוד | `secret-word` | קריאת זוג סדור (x,y) | המילה **נקודה** | ✓ |
| ציור נסתר | `hidden-drawing` | סימון וחיבור זוגות סדורים | ציור **בית** (מסלול סגור ברביע הראשון) | ✓ |
| המסלול המוצפן | `encrypted-route` | הזזה (ימינה/שמאלה/למעלה/למטה) | המילה **קסם** | ✓ |
| הנקודה החשודה | `suspect-point` | ניתוח שיעורים ואי־שוויונים | המילה **אמת** (נקודה יחידה לכל רמזים) | ✓ |
| אותו x או אותו y | `same-axis` | שיעורים זהים וקווים מקבילים לצירים | המילה **ישר** | ✓ |
| כספת הקואורדינטות | `coordinate-safe` | שיעורים, הזזה ונקודות על הצירים | הקוד **4705** | ✓ |
| מבוך הקואורדינטות | `coordinate-maze` | הזזה מדויקת עד ליעד | הגעה מ־(0,0) ל־(6,4) | ✓ (BFS מוכיח פתירוּת) |
| פענוח צבעוני | `color-decode` | סימון מדויק של זוגות סדורים | סמל **וי** | ✓ |

## כללים שנשמרו

- אין כותרות "שעשועון 1/2" ואין תוויות "קל/בינוני/קשה".
- אין בקשה מהילדים להמציא משחק או תשובה.
- כל תשובה נכונה אחת בלבד היכן שנדרשת תשובה יחידה (מאומת בבדיקות: `suspectsOf(round)` בדיוק אחד; מילות סוד מתקבלות רק מהתשובות הנכונות).
- כל נקודה ברביע הראשון ובתחום הגרף (מאומת: `inRange`).

## הרחבה

להוספת שעשועון: צרו `GameDefinition` תחת `src/games/`, הפרידו את הנתונים מהתצוגה, רשמו ב־`GAMES`, והוסיפו בדיקה שמוכיחה פתירוּת ותוצאה נכונה.

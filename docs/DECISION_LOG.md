## שיטת עבודה פשוטה — החלטה מחייבת
- יניב שולח את דרישות העריכה בעברית רגילה, מכל מכשיר.
- העריכות מתבצעות ישירות באמצעות GitHub כברירת מחדל.
- אין לבקש מיניב להוריד ZIP, להריץ PowerShell, CMD או Python לצורך תיקונים רגילים.
- כמה דרישות קשורות מרוכזות לעדכון אחד מסודר ככל האפשר.
- בסיום נמסרים רק השינויים, תוצאות הבדיקה, ה־commit וכתובת התצוגה.
- אין ליצור workflows זמניים, סקריפטים חד־פעמיים או מסלולי התקנה מורכבים כשיש דרך ישירה.

## החלטות פעילות — נכון/לא נכון, הזזה והדפסה
- בכל מקבץ נכון/לא נכון מופיעות גם תשובות נכונות וגם תשובות לא נכונות.
- בטבלה בת ארבע שורות ומעלה נשמר איזון קרוב ככל האפשר בין שני סוגי התשובות.
- לכל שורה יש שתי תיבות סימון אחידות: ליד „נכון” וליד „לא נכון”.
- סדר התשובות מגוון ואינו חוזר תמיד על אותה תבנית.
- בהקשר של נקודות משתמשים ב„זזים”, „מזיזים” ו„הזזה”; אין להשתמש ב„מתקדמים”.
- קיימות תצוגה והדפסה חכמות בשחור־לבן, בגווני אפור מובחנים ובניגודיות גבוהה.

## החלטות פעילות — ארכיטקטורה מאוחדת (2026-07-20)
- הפרויקט אוחד לאפליקציה אחת מבוססת Vite + TypeScript: ריפו אחד, כתובת אחת, `index.html` אחד.
- כל התוכן עצמאי בתוך הריפו; אין תלות ברשת בזמן ריצה ואין iframe אל parabula-next.
- 8 השעשועונים משולבים באפליקציה כרכיבים מתמטיים מדויקים ונבדקים.
- נוספו בדיקות אוטומטיות (Vitest) למתמטיקה, למשחקים ולתוכן החובה, ו־CI לבנייה/בדיקה.
- אזור למורה ודפי פתרונות אינם בהיקף הפרויקט (לפי הנחיית יניב).

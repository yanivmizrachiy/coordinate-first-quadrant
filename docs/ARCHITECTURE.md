# אדריכלות

## עקרונות

- **אפליקציה אחת, ריפו אחד, כתובת אחת, קובץ כניסה אחד** (`index.html`).
- **אתר סטטי** — ללא שרת. עובד ב־GitHub Pages ובכל אירוח סטטי.
- **הפרדת נתונים מתצוגה** — תוכן העמודים ונתוני המשחקים הם מבני נתונים; התצוגה מרנדרת אותם.
- **ללא ספריות כבדות** — TypeScript ו־DOM בלבד, בלי framework זמן־ריצה. משקל קטן, שליטה מלאה, דיוק הדפסה.
- **הכול נבדק** — המתמטיקה, פתירוּת המשחקים ותוכן החוברת מכוסים בבדיקות.

## שכבות

```
index.html
  └─ src/main.ts            אתחול: סרגל עליון, outlet, ניתוב, מצב שחור־לבן
       ├─ router.ts         נתב מבוסס hash: #/  #/workbook/:n  #/games/:id …
       ├─ views/            כל מסך בונה DOM ומחזיר פונקציית ניקוי
       ├─ data/workbook/    34 עמודים (WorkbookPageContent[]) + TOPICS
       ├─ games/            8 משחקים (GameDefinition) + revealEngine משותף
       ├─ lib/coordinateGrid.ts   מרנדר SVG של מערכת צירים ברביע הראשון
       ├─ lib/coordinateMath.ts   פונקציות טהורות (נבדקות) לכל האינוריאנטות
       ├─ lib/dom.ts / storage.ts עזרי DOM ו-localStorage ממודרים
       └─ styles/           tokens → base → app → workbook → grayscale
```

## ניתוב

נתב מבוסס `hash` נבחר במכוון: הוא עובד באירוח סטטי ללא הגדרות שרת ותומך בקישור ישיר לכל עמוד/משחק. `main.ts` שומר פונקציית ניקוי אחת ומריץ אותה במעבר בין מסכים כדי למנוע דליפות מאזינים.

## מודל הגרף

`renderCoordinateGrid(spec)` מקבל `GridSpec` מטופס (points, segments, arrows, polygons, labelboxes, xlabels/ylabels) ומחזיר `<svg>` וקטורי חד. הגאומטריה זהה לחוברת המקורית (viewBox 520×330, ציר x עד 8, ציר y עד 6). עמודי החוברת שומרים על ה־`<div class="coordinate-grid" data-…>` המקורי, ו־`hydrateGrids()` ממיר אותם ל־SVG בזמן טעינת המסך.

## תוכן החוברת

30 עמודי הליבה (`src/data/workbook/legacy-pages.ts`) הוגרו פעם אחת מהחוברת המקורית. במהלך ההגירה נאפו שלושת התיקונים שהיו בעבר בזמן־ריצה בדפדפן:
1. ניסוח תנועה (`מתקדמים` → `זזים/הזזה`).
2. איזון טבלאות נכון/לא־נכון + תיבות סימון.
3. תיבות תשובה לרשת הטעויות בעמוד 25.

מאז ההגירה הושלמה, וכלי ההגירה החד־פעמי הוסר. התוכן מתוחזק כעת כקוד מקור נערך: לתיקון עמוד עורכים את מחרוזת ה־HTML שלו. עמודים 31–34 נכתבים ידנית ב־`continuation.ts`. החוברת המקורית שמורה בהיסטוריית git (commit `a9087db7`) לשחזור במקרה הצורך.

## מנוע המשחקים המשותף

`revealEngine.ts` מפעיל את משחקי "פתרו משימות ← גלו מילה/קוד" (מילת הסוד, כספת הקואורדינטות, המסלול המוצפן). בדיקת התשובות (`normalizeAnswer`, `isStepCorrect`, `solutionOf`) טהורה ומיוצאת לבדיקות, כך שאפשר להוכיח שכל חידה פתירה ושהתשובות הנכונות מרכיבות בדיוק את התוצאה המכוונת. שאר המשחקים (אותו x/y, הנקודה החשודה, ציור נסתר, מבוך, פענוח צבעוני) בנויים כרכיבים ייעודיים, אך גם הם מפרידים נתונים מתצוגה.

## עיצוב

מערכת עיצוב מבוססת CSS custom properties (`styles/tokens.css`) — צבעים, טיפוגרפיה, מרווחים ורדיוסים. `workbook.css` הוא נמל נאמן של סגנונות החוברת המקורית, כדי ש־34 העמודים ייראו זהים. `grayscale.css` מוסיף מצב שחור־לבן איכותי לתצוגה ולהדפסה.
